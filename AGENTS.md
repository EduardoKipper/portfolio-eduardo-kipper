# Regras para agentes e colaboradores

Estas regras se aplicam a todo o repositório:

- Use React com TypeScript.
- Utilize apenas componentes funcionais.
- Não utilize Tailwind CSS.
- Não utilize estilos inline.
- Use CSS Modules para os estilos de componentes.
- Restrinja o CSS global a reset, tokens de design e utilitários realmente globais.
- Não adicione bibliotecas sem uma justificativa técnica clara e registrada.
- Mantenha os componentes pequenos, coesos e reutilizáveis.
- Use HTML semântico e escolha os elementos por seu significado, não por sua aparência.
- Mantenha acessibilidade básica: navegação por teclado, foco visível, rótulos, textos alternativos e contraste adequado.
- Não invente textos profissionais definitivos, incluindo cargos, clientes, datas, resultados, contatos ou competências.
- Use conteúdo temporário somente quando necessário e identifique-o claramente, por exemplo com `[CONTEÚDO TEMPORÁRIO]`.
- Execute `npm run lint` e `npm run build` antes de concluir qualquer alteração.
- Não modifique arquivos de documentos ou imagens fornecidos, incluindo os arquivos em `docs/references/`, `public/documents/` e `public/images/`.
